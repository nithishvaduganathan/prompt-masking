================================================================================
IMPLEMENTATION OVERVIEW
================================================================================

This project implements a complete end-to-end privacy-preserving AI chatbot
system that automatically masks sensitive personal information before sending
queries to external Large Language Models (LLMs), then unmasks the responses
locally to provide natural user-friendly answers.

================================================================================
KEY COMPONENTS
================================================================================

1. MASKING MODULE (app/masking/masker.py)
   - 360+ lines of advanced NLP masking logic
   - Detects 8+ categories of sensitive data:
     * Mental health conditions (20+ terms)
     * Medical diseases (25+ conditions)
     * Email addresses (RFC-compliant patterns)
     * Phone numbers (multiple formats)
     * Age information (various patterns)
     * Gender information
     * Location data (100+ locations)
     * Names (optional spaCy NER)
   - Generates unique placeholder tokens: [MENTAL_HEALTH_0], [EMAIL_0], etc.
   - Reversible unmasking with mapping preservation

2. FLASK REST API (app/routes/main.py)
   - /api/chat - Complete privacy-preserving chat flow
   - /api/mask - Standalone masking service
   - /api/unmask - Standalone unmasking service
   - /api/health - Service health monitoring
   - Comprehensive error handling
   - Server-side session management

3. LLM INTEGRATION (app/llm_client.py)
   - Simulated intelligent responses (default)
   - OpenAI GPT-3.5-turbo support

   - Automatic fallback on errors

HOW IT WORKS
================================================================================

1. USER INPUT
   User enters: "I'm 25 with anxiety. Email: test@example.com"

2. LOCAL MASKING
   System detects: AGE, MENTAL_HEALTH, EMAIL
   Replaces with: "I'm [AGE_0] with [MENTAL_HEALTH_0]. Email: [EMAIL_0]"
   Stores mapping: {[AGE_0]: "25", [MENTAL_HEALTH_0]: "anxiety", ...}

3. LLM QUERY
   Only masked prompt sent to external AI service
   Sensitive data NEVER leaves local environment

4. LLM RESPONSE
   Returns: "At [AGE_0], [MENTAL_HEALTH_0] is common. Contact us at..."

5. LOCAL UNMASKING
   System restores: "At 25, anxiety is common. Contact us at..."

6. USER DISPLAY
   Natural response shown with transparency details
